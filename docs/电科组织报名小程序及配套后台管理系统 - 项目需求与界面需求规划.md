# 电科组织报名小程序及配套后台管理系统 - 项目需求与界面需求规划

# 一、项目概述

## 1.1 项目目标

为高校社团电子科技小组开发轻量化快捷报名小程序及配套后台管理系统，实现学生线上便捷报名、个人信息管理，以及管理员对报名数据、用户数据的集中管控、统计分析等功能，提升社团报名管理的效率与规范性。

## 1.2 核心用户

- 小程序端：高校电子科技小组成员及意向报名学生（以下统称“学生用户”）

- 后台管理系统端：社团管理员（单一管理员账号）

## 1.3 技术栈要求

- 小程序端：Uni-app + TypeScript / Taro + Vue3 + TypeScript（轻量化开发，优先保证核心功能流畅）

- 后台管理系统端：React19 + React Compiler + TypeScript + Ant Design v6（标准后台布局，注重数据管理与统计功能）

# 二、小程序端需求规划

## 2.1 核心业务流程

用户进入小程序 → 未登录则跳转至注册/登录页 → 注册（学号为账号，自定义密码，账号唯一）/登录后 → 进入首页（TabBar布局）→ 报名Tab：查看当前批次报名状态（未报名则显示“快速报名”按钮，已报名则显示“报名记录”入口）→ 个人中心Tab：进行密码修改、头像设置、查看报名结果、反馈等操作。

## 2.2 功能需求

### 2.2.1 登录与注册模块

- 登录功能：
        

    - 输入项：学号（账号）、密码

    - 验证规则：账号非空、密码非空、账号与密码匹配校验

    - 异常处理：账号不存在提示、密码错误提示（错误次数限制可选，建议3次后短暂锁定）

    - 记住密码：提供“记住密码”勾选框（本地存储，非敏感信息加密处理）

    - 跳转逻辑：登录成功后跳转至首页；未登录状态下访问任何页面均拦截跳转至登录/注册页

- 注册功能：
        

    - 输入项：学号（账号）、密码、确认密码、姓名（可选，可后续在个人中心完善）

    - 验证规则：
                

        - 学号：非空、格式校验（如匹配高校学号规则，例：2023XXXXXX）、唯一性校验（实时查询后端，提示“该学号已注册”）

        - 密码：非空、长度限制（建议6-20位）、支持字母+数字组合（增强安全性）

        - 确认密码：与密码一致校验

    - 注册成功：自动登录并跳转至首页，同时提示“注册成功”

### 2.2.2 首页（TabBar容器）

- TabBar配置：2个Tab，固定在底部，切换时高亮当前Tab
        

    - 第一个Tab：报名（图标建议：表单/报名相关图标）

    - 第二个Tab：个人中心（图标建议：用户/个人相关图标）

- 页面状态：仅登录后可见，未登录拦截至登录/注册页

### 2.2.3 报名Tab页面

- 当前批次报名状态查询：进入页面后自动请求后端，查询当前登录用户在最新报名批次的报名状态

- 未报名状态：
        

    - 显示当前报名批次信息（如：电子科技小组202X年秋季招新）、报名截止时间、报名须知（简洁文本）

    - 突出显示“快速报名”按钮（按钮样式醒目，居中或靠上位置）

    - 点击“快速报名”跳转至报名表单填写页

- 已报名状态：
        

    - 显示“已报名”状态标签（绿色醒目样式）

    - 显示报名批次、报名时间、报名状态（待审核/已通过/未通过）

    - 显示“查看报名记录”入口，点击跳转至报名记录列表页

- 无当前报名批次：显示“暂无正在进行的报名批次”提示文本，无操作按钮

### 2.2.4 报名表单填写页

- 表单字段（核心必填，简化设计）：
        

    - 基础信息：姓名（自动带出注册时姓名，可修改）、学号（不可修改，只读）、专业、年级、班级、联系电话（必填，验证手机号格式）

    - 报名相关：报名意向（如：硬件开发/软件开发/电子竞赛，单选或多选，后端可配置）、个人简介（可选，文本域，限制字数）

- 表单验证：
        

    - 必填项非空校验

    - 联系电话格式校验（11位手机号）

    - 提交前整体校验，未通过则提示具体错误项

- 操作按钮：
        

    - 取消：点击返回报名Tab页面，提示“是否放弃填写？”

    - 提交：点击后提交表单至后端，提交成功提示“报名成功”，并返回报名Tab页面（此时显示已报名状态）；提交失败提示具体原因（如：网络错误、后端异常）

### 2.2.5 报名记录列表页

- 列表展示：按报名批次倒序排列（最新批次在前），每一项包含：报名批次名称、报名时间、报名状态（待审核/已通过/未通过，不同状态不同颜色标签）

- 操作：点击某条记录，跳转至报名记录详情页

- 空状态：无报名记录时，显示“暂无报名记录”提示文本及相关图标

### 2.2.6 报名记录详情页

- 展示内容：报名批次、报名时间、报名状态、所有表单填写的信息（结构化展示，清晰易读）

- 结果反馈：若状态为“已通过”/“未通过”，显示管理员填写的反馈意见（可选，后端可配置是否需要反馈）

- 操作按钮：返回，点击返回报名记录列表页

### 2.2.7 个人中心Tab页面

- 个人信息展示区：


    - 头像：圆形展示，默认使用占位头像（如学号首字母/默认图标）

    - 基本信息：姓名、学号（账号）、专业（若已完善）

    - 操作：点击头像或信息区域，跳转至个人信息编辑页

- 功能入口（列表式，带图标）：
        

    - 报名记录：点击跳转至报名记录列表页（与报名Tab的报名记录入口一致）

    - 密码设置：点击跳转至密码修改页

    - 用户反馈：点击跳转至反馈提交页

    - 关于我们：点击跳转至关于我们页

    - 联系我们：点击跳转至联系我们页

    - 版本信息：点击展示当前小程序版本号

    - 退出登录：点击后提示“是否确认退出登录？”，确认后清除本地登录状态，跳转至登录/注册页

### 2.2.8 个人信息编辑页

- 头像设置：

    - 展示当前头像，提供“更换头像”按钮

    - 头像来源：从手机相册选择/拍照（小程序授权）

    - 头像裁剪：支持简单裁剪（如正方形裁剪，保证头像比例）

- 基本信息编辑：姓名、专业、年级、班级（可修改，提交后同步至后端）

- 操作按钮：
        

    - 取消：放弃修改，返回个人中心页

    - 保存：提交修改内容，保存成功提示“修改成功”，返回个人中心页（更新展示信息）

### 2.2.9 密码修改页

- 输入项：原密码、新密码、确认新密码

- 验证规则：
       

    - 原密码：非空，与当前登录用户密码匹配校验

    - 新密码：非空，长度6-20位，字母+数字组合，不与原密码一致

    - 确认新密码：与新密码一致校验

- 操作按钮：
        

    - 取消：返回个人中心页

    - 保存：验证通过后提交修改，保存成功提示“密码修改成功，请重新登录”，并清除本地登录状态，跳转至登录页；验证失败提示具体错误

### 2.2.10 反馈提交页

- 输入项：反馈类型（下拉选择，如：功能异常、建议优化、其他）、反馈内容（文本域，必填，限制字数）、联系方式（可选，手机号/微信号，便于后续沟通）

- 操作按钮：
        

    - 取消：返回个人中心页

    - 提交：提交反馈至后端，提交成功提示“反馈提交成功，感谢您的建议！”，返回个人中心页

### 2.2.11 关于我们页

展示内容：电子科技小组简介、成立时间、核心宗旨、主要活动（文本+简单图片，图片可选，轻量化可暂不添加）

### 2.2.12 联系我们页

展示内容：社团负责人姓名、联系电话（可点击拨打）、QQ群号（可点击复制）、办公地点（可选）

### 2.2.13 版本信息页

展示内容：当前版本号（如：v1.0.0）、更新时间、更新说明（简洁文本，如：初始版本，实现快捷报名功能）

## 2.3 界面需求

### 2.3.1 整体风格

轻量化、简洁清晰、易用性优先，色调建议采用科技蓝为主色调（契合电子科技主题），辅助色采用浅灰色，文字色为深灰色（保证可读性），避免复杂动画和冗余装饰。

### 2.3.2 页面布局规范

- 所有页面统一设置顶部导航栏，包含标题（如：登录、报名、个人中心）、返回按钮（非首页页面）

- 表单页面采用纵向布局，输入项上下排列，标签在左/上，输入框在右/下，间距统一（建议16px）

- 列表页面采用卡片式布局，每一项独立卡片，包含核心信息，点击区域明确

- 按钮样式统一：主要按钮（提交、确认）采用主色调背景，白色文字；次要按钮（取消）采用白色背景，主色调边框和文字

### 2.3.3 关键页面界面细节

- 登录/注册页：
        

    - 居中布局，表单区域占页面宽度70%-80%，上下留有足够间距

    - 提供“已有账号？去登录”/“没有账号？去注册”切换入口，位于表单底部

    - 输入框带清除按钮（输入内容后显示），密码框带显示/隐藏密码切换按钮

- 报名Tab页面：
        

    - 报名状态标签位置靠上，醒目可见

    - “快速报名”按钮尺寸较大，位置居中或靠上，避免用户寻找

    - 报名须知文本采用浅灰色，字体稍小，避免干扰核心操作

- 个人中心页：
        

    - 个人信息展示区背景可采用浅主色调，与下方功能入口区分开

    - 功能入口列表每一项带左侧图标，右侧带箭头（提示可点击跳转）， hover状态有背景色变化

    - “退出登录”入口位于列表底部，文字颜色为红色（警示色）

# 三、后台管理系统需求规划

## 3.1 核心业务流程

管理员进入系统 → 登录页面（输入管理员账号密码）→ 登录成功后进入系统首页（仪表板）→ 通过左侧侧边栏切换各功能模块（报名记录管理、用户管理、数据统计、系统设置）→ 执行对应操作（查询、导出、审核、设置等）→ 退出登录（清除登录状态，返回登录页）。

## 3.2 功能需求

### 3.2.1 登录模块

- 无注册功能，仅支持单一管理员账号登录

- 输入项：用户名（固定管理员账号，可在系统设置中修改）、密码

- 验证规则：用户名非空、密码非空、用户名与密码匹配校验

- 异常处理：密码错误提示“用户名或密码错误”，登录失败次数限制（可选，3次后短暂锁定）

- 记住密码：提供“记住密码”勾选框（后端存储登录状态，设置有效期）

- 登录成功：跳转至系统首页（仪表板）；登录失败：停留在登录页，显示错误提示

### 3.2.2 系统布局（标准Layout）

- Header（顶部导航）：

    - 左侧：系统名称（电子科技小组报名管理系统）

    - 右侧：管理员账号信息（如“管理员”）、退出登录按钮

- SiderBar（左侧侧边栏）：
        

    - 展示所有功能菜单：仪表板、报名记录管理、用户管理、数据统计、系统设置

    - 当前激活菜单高亮显示（主色调背景或文字）

    - 支持折叠/展开（响应式，屏幕较小时自动折叠，hover展开）

- Content（中间内容区）：
        

    - 展示当前选中菜单的对应页面内容

    - 页面统一设置面包屑导航（如：首页 → 报名记录管理），便于定位当前位置

- Footer（底部页脚）：
        

    - 展示版权信息（如：© 202X 电子科技小组 版权所有）、系统版本号

    - 居中对齐，样式简洁

### 3.2.3 仪表板模块

- 数据概览（卡片式展示）：
       

    - 总用户数：已注册学生用户总数

    - 当前批次报名数：最新报名批次的报名人数

    - 待审核数：所有批次处于“待审核”状态的报名记录数

    - 已通过数：所有批次处于“已通过”状态的报名记录数

- 近期报名趋势（图表）：
        

    - 采用折线图/柱状图，展示近3-5个报名批次的报名人数变化趋势

    - 支持切换图表类型（折线/柱状），可查看具体数值

- 最新报名记录（列表）：
        

    - 展示最近10条报名记录，包含：学号、姓名、报名批次、报名时间、报名状态

    - 点击记录可跳转至报名记录详情页（审核页面）

### 3.2.4 报名记录管理模块

- 报名记录查询：
        

    - 多条件筛选：报名批次（下拉选择，后端可配置）、报名状态（待审核/已通过/未通过，多选）、学号/姓名（模糊搜索）、报名时间范围（日期选择器）

    - 查询结果实时展示，支持分页（每页10/20/50条，可切换）

- 报名记录列表：
        

    - 展示字段：序号、学号、姓名、专业、报名批次、报名时间、报名状态、操作

    - 操作列：查看/审核（点击跳转至详情/审核页）、删除（可选，需二次确认）

- 报名审核功能：
       

    - 审核页面展示报名学生的所有表单信息

    - 审核操作：选择审核结果（通过/未通过）、填写审核意见（可选，未通过时建议必填）

    - 提交审核：点击“提交审核”后，更新报名记录状态，同步至小程序端

- Excel导出功能：
        

    - 提供“导出当前页”/“导出全部筛选结果”两种导出选项

    - 导出字段可配置（默认导出所有列表字段+表单详情字段）

    - 导出文件命名规范：报名记录_报名批次_导出时间（如：报名记录_202X秋季招新_202X1001.xlsx）

    - 导出过程中显示加载提示，导出完成自动下载文件

- 报名批次管理（可选，集成在此模块）：
        

    - 支持新增/编辑/删除报名批次（如：设置批次名称、报名开始时间、截止时间、报名须知）

    - 支持设置当前活跃批次（小程序端仅展示当前活跃批次的报名状态）

### 3.2.5 用户管理模块

- 用户查询：
        

    - 多条件筛选：学号（精确/模糊）、姓名（模糊）、专业（下拉选择）、注册时间范围（日期选择器）

    - 查询结果分页展示，支持分页大小切换

- 用户列表：
        

    - 展示字段：序号、学号（账号）、姓名、专业、年级、注册时间、操作

    - 操作列：查看详情（展示用户完整信息）、重置密码（重置为默认密码，如123456，重置后提示管理员告知用户）、删除（可选，需二次确认）

- Excel导出功能：


    - 支持“导出当前页”/“导出全部筛选结果”

    - 导出字段：学号、姓名、专业、年级、注册时间、联系电话（若已完善）

    - 文件命名规范：用户列表_导出时间.xlsx

### 3.2.6 数据统计模块

- 用户数据统计：

    - 按专业/年级统计用户分布（饼图/柱状图）

    - 注册时间趋势（折线图，展示每日/每周注册人数）

- 报名数据统计：
        

    - 各批次报名人数统计（柱状图，对比不同批次报名热度）

    - 各报名意向分布（饼图，如硬件开发/软件开发人数占比）

    - 各批次审核通过率统计（折线图/柱状图）

- 统计数据导出：支持将各统计图表对应的原始数据导出为Excel

- 数据筛选：支持按时间范围（如近1个月/近3个月/自定义）筛选统计数据

### 3.2.7 系统设置模块

- 管理员账号设置：
        

    - 修改管理员用户名（默认admin，可自定义）

    - 修改管理员密码（需输入原密码、新密码、确认新密码，验证规则同小程序用户密码）

- 小程序配置：
        

    - 配置小程序报名表单字段（如是否显示“个人简介”字段、报名意向选项增减）

    - 配置报名须知文本（支持富文本编辑，同步至小程序端）

    - 配置“关于我们”“联系我们”内容（同步至小程序端）

- 系统参数设置：
        

    - 设置登录失败锁定时间（如5分钟）

    - 设置Excel导出最大数据量（如1000条）

- 用户反馈管理：
        

    - 展示小程序用户提交的所有反馈，按提交时间倒序排列

    - 展示字段：提交人（学号/匿名）、反馈类型、反馈内容、联系方式、提交时间、处理状态（未处理/已处理）

    - 操作：标记为已处理、删除反馈

### 3.2.8 退出登录功能

# 四、公共需求

## 4.1 数据交互需求

- 接口规范：前后端采用RESTful API风格，统一请求头（如携带token）、响应格式（如{code: 0, msg: "成功", data: {}}）

- 数据校验：前端做基础校验（非空、格式），后端做全面校验（包含权限、数据合法性），避免恶意请求

- 异常处理：网络异常、接口报错时，前端显示友好提示（如“网络错误，请稍后重试”“系统异常，请联系管理员”），后端记录错误日志

- 数据同步：小程序端与后端数据实时同步（如报名状态、审核结果），后台修改配置后即时同步至小程序

## 4.2 性能需求

- 小程序端：页面加载时间≤2秒，表单提交响应时间≤1秒，适配不同型号手机（iOS/Android，屏幕尺寸4.7-6.7英寸）

- 后台管理系统：页面加载时间≤3秒，列表查询（数据量≤1000条）响应时间≤1秒，Excel导出（数据量≤1000条）时间≤5秒，支持主流浏览器（Chrome、Edge、Firefox最新版本）

## 4.3 安全需求

- 密码安全：用户密码、管理员密码采用加密存储（如MD5+盐值），传输过程中加密（HTTPS）

- 权限控制：小程序端仅能访问当前登录用户的数据，后台管理系统仅管理员可登录，不同功能模块权限明确（如仅系统设置模块可修改管理员信息）

- 防恶意请求：后端对接口访问频率进行限制（如同一IP/账号每分钟最多请求60次），防止爬虫和恶意攻击

- id：主键ID（int，自增）
